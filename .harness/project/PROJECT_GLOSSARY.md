# Project Glossary

Last reviewed: 2026-05-12

- AI: automated player controller used by `P2` by default.
- Attack target: either an opponent monster or opponent core, represented by `AttackTarget`.
- Board cell: a single grid position with optional dungeon tile, core, owner, and monster occupancy.
- Core / Heart Core: player base objective. Reducing opponent core HP to zero wins the game.
- Crest: resource generated by dice rolls. Current crest types are `summon`, `move`, `attack`, `defense`, `magic`, and `trap`.
- Dice catalog: available dice definitions in `src/game/data/diceCatalog.ts`.
- Dungeon network: connected player-owned core/tile network used as an anchor for summon placement.
- Dungeon tile: placed board tile that enables movement paths and summon expansion.
- Game phase: one of `roll`, `summon`, `action`, or `gameOver`.
- Interaction mode: UI mode for `placing`, `moving`, `attacking`, or `none`.
- Monster definition: static monster stats from `src/game/data/monsters.ts`.
- Monster instance: live summoned monster on the board.
- Tile shape: shape offsets used for dungeon placement.
